Setting Up My Bash for Efficiency

Today I am setting up my bash shell. While I don’t have a lot of experience with it, I'm working hard to optimize my workflow and make things easier. Part of me wanted to stick with the default bash configuration, but after a while, I realized that some commands are just too long, and it's worth finding shortcuts.

Step 1: Backup Your Bash

Before making any changes, it’s important to create a backup of your bash configuration in case things go wrong. Here's how to find the bash location and create a backup: 


        $ which bash
        $ sudo cp /bin/bash ~/bash-backup

To verify the backup:


        $ ls -l ~/bash-backup

If you ever need to restore the backup, use:


        $ sudo cp ~/bash-backup /bin/bash

Step 2: Edit the Bash Configuration

Next, let's make some improvements to the bash configuration file by editing ~/.bashrc. This file contains the aliases and functions we will modify. 


        $ nano ~/.bashrc

Step 3: Add Useful Aliases

Now, let's add some aliases to streamline common tasks. Aliases are shortcuts for frequently used commands. Below are some useful ones to get started: 


        # Utility Aliases
        alias q='exit'
        alias c='clear'
        alias h='history'
        alias cs='clear; ls'
        alias p='cat'
        alias pd='pwd'
        alias lsa='ls -a'
        alias lsl='ls -l'
        alias t='time'
        alias k='kill'
        alias null='/dev/null'

        # Directory Shortcuts
        alias home='cd ~'
        alias root='cd /'
        alias dtop='cd ~/Desktop'
        alias dbox='cd ~/Dropbox'
        alias gdrive='cd ~/Google\ Drive'

        # Project Directories
        alias cppprojects='cd ~/Dropbox/Projects/C++Projects'
        alias pythonprojects='cd ~/Dropbox/Projects/PythonProjects'
        
        # Git Aliases
        alias g='git'
        alias st='git status'
        alias com='git commit -m'
        alias clone='git clone'
        alias lg='git log'

        # Navigation Shortcuts
        alias ..='cd ..'
        alias ...='cd ../..'
        alias ....='cd ../../..'

Step 4: Apply and Commit Changes

To apply the changes, reload the .bashrc file: 


        $ source ~/.bashrc

Once everything is set up, you can quickly commit your changes to Git:


        $ u
        $ com 'updated bashrc with useful aliases'

Final Thoughts

Setting up these aliases and shortcuts is a great way to improve efficiency, especially with frequently used commands. It's a small change but makes a big difference in day-to-day tasks. I highly recommend keeping a copy of your aliases handy, whether digitally or printed, as well as expanding your knowledge of other useful tools like VIM or custom scripts.
