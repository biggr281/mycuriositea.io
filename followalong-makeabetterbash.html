Setting Up My Bash

Today, I'm working on setting up my bash configuration. While I don’t have much experience, I'm determined to work smarter, not harder! Though part of me wants to stick with the default bash, some commands are just too long. Here's what I've learned so far.

Backing Up the Original Bash

Before making any changes, it's important to back up the original bash configuration in case something goes wrong. Here's how you can do that: 


        $ which bash
        $ sudo cp /bin/bash ~/bash-backup
        $ ls -l ~/bash-backup

If something goes wrong, you can restore the original bash with this command: 


        $ sudo cp ~/bash-backup /bin/bash

Common Aliases

I’ve set up some common aliases to simplify my workflow. These save time by shortening frequently used commands: 


        $ alias q='exit'
        $ alias c='clear'
        $ alias h='history'
        $ alias cs='clear; ls'
        $ alias p='cat'
        $ alias pd='pwd'
        $ alias lsa='ls -a'
        $ alias lsl='ls -l'
        $ alias t='time'
        $ alias k='kill'

Directory Navigation Aliases

For easier directory navigation, I've set up these aliases: 


        $ alias home='cd ~'
        $ alias root='cd /'
        $ alias dtop='cd ~/Desktop'
        $ alias dbox='cd ~/Dropbox'
        $ alias cppprojects='cd ~/Dropbox/Projects/C++Projects'
        $ alias pythonprojects='cd ~/Dropbox/Projects/PythonProjects'
        $ alias gdrive='cd ~/Google\\ Drive'

Git Shortcuts

Git is essential for version control, and these aliases make working with git easier: 


        $ alias g='git'
        $ alias st='git status'
        $ alias com='git commit -m'
        $ alias clone='git clone'
        $ alias lg='git log'
        $ alias u='git add -u'
        $ alias all='git add .'

Additional Aliases

Here are a couple of additional handy shortcuts: 


        $ alias myec2box='ssh '
        $ alias pythonserver='python -m SimpleHTTPServer 8000'

Final Thoughts

These aliases streamline my workflow and reduce the errors caused by longer commands. I've also learned the importance of keeping a digital and printed copy of useful shortcuts. For now, this is a solid start, but I plan to expand my setup as I learn more!
